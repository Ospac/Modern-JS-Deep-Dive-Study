# 18장 함수와 일급 객체
## 18.1 일급 객체
다음의 조건을 갖춘 객체를 **일급 객체**라 한다.
1. 무명의 리터럴로 생성할 수 있다. (런타임에 생성이 가능하다)
2. 변수나 자료구조(객체, 배열등)에 저장할 수 있다.
3. 함수의 매개변수에 전달할 수 있다.
4. 함수의 반환값으로 사용할 수 있다.

자바스크립트의 함수는 일급객체다.
- 객체는 값이므로 함수도 값과 동일하게 취급된다.
- 변수 할당문, 객체의 프로퍼티 값, 배열의 요소, 인수, 반환문등에서 함수가 사용될 수 있다. 이러한 특징은 함수형 프로그래밍을 가능하게 한다. (함수의 조합으로)
- 함수 객체는 고유의 프로퍼티가 있다.
```js
// 1. 함수는 무명의 리터럴로 생성할 수 있다.
// 2. 함수는 변수에 저장할 수 있다.
// 런타임(할당 단계)에 함수 리터럴이 평가되어 함수 객체가 생성되고 변수에 할당된다.
const increase = function (num) {
  return ++num;
};

// 2. 함수는 객체에 저장할 수 있다.
const auxs = { increase };

// 3. 함수의 매개변수에게 전달할 수 있다.
// 4. 함수의 반환값으로 사용할 수 있다.
function makeCounter(aux) {
  let num = 0;

  return function () {
    num = aux(num);
    return num;
  };
}

// 3. 함수는 매개변수에게 함수를 전달할 수 있다.
const increaser = makeCounter(auxs.increase);
console.log(increaser()); // 1
console.log(increaser()); // 2

```
## 18.2 함수 객체의 프로퍼티 
함수 객체는 `arguments`, `caller`, `length`, `name`, `prototype`등의 고유의 프로퍼티를 가진다.(`__proto__`는 모든 객체가 상속받는 프로퍼티)

### 18.2.1 arguments 프로퍼티
`arguments` 프로퍼티 값은 `arguments`객체다.
- 함수 호출 시 전달된 인수들의 정보를 담고 있는 순회 가능한<sup>iterable</sup> 유사 배열 객체
- 함수 내부에서 지역 변수처럼 사용된다. 외부에서 참조할 수 없다

자바스크립트는 선언된 매개변수의 개수와 전달하는 인수의 개수의 일치를 확인하지 않는다.
- 초과하여 전달된 인수를 포함한 모든 인수는 `arguments` 객체에 저장된다.
- 적게 전달받아 할당되지 않는 매개변수는 암묵적으로 `undefined`가 할당된다.

따라서 `arguments` 객체는 매개변수 개수를 확정할 수 없는 **가변 인자 함수**를 구현할때 유용하다.
```js
function sum() {
  let res = 0;

  // arguments 객체는 length 프로퍼티가 있는 유사 배열 객체이므로 for 문으로 순회할 수 있다.
  for (let i = 0; i < arguments.length; i++) {
    res += arguments[i];
  }

  return res;
}

console.log(sum());        // 0
console.log(sum(1, 2));    // 3
console.log(sum(1, 2, 3)); // 6
```

`arguments` 객체는 배열 형태로 인자 정보를 담고있지만, 실제 배열이 아닌 `유사 배열 객체`이다. (`length` 프로퍼티를 가져서 for문으로 순회할 수 있는 객체)

> ES6에서 도입된 이터레이션 프로토콜(34.1)을 준수하면 순회 가능한 자료구조인 이터러블이 된다. `arguments`는 유사 배열 객체이면서 이터러블이다.

- 따라서 `arguments`는 배열 메서드를 사용할 수 없다.
- 이러한 단점 때문에, ES6에서는 `Rest` 파라미터를 도입하여 `arguemnts` 객체를 대신한다. (26.4)

### 18.2.2 caller 프로퍼티

비표준 프로퍼티, 함수 자신을 호출한 함수를 가리킴 (중요하지 않아 생략, 예시 참고)

### 18.2.3 length 프로퍼티
함수를 정의할 때 선언한 매개변수의 개수를 가리킨다.
- 전달된 인수를 저장하는 `arguments` 객체의 `length`와는 값이 다를 수 있다.

```js
function foo() {}
console.log(foo.length); // 0

function bar(x) {
  return x;
}
console.log(bar.length); // 1

function baz(x, y) {
  return x * y;
}
console.log(baz.length); // 2
```

### 18.2.4 name 프로퍼티
ES6에서 표준이 되었다. 함수의 이름을 값으로 가진다.
- 함수의 이름이 없을때 ES5와 ES6에서 동작이 다르다.
```js
// 기명 함수 표현식
var namedFunc = function foo() {};
console.log(namedFunc.name); // foo

// 익명 함수 표현식
var anonymousFunc = function() {};
// ES5: name 프로퍼티는 빈 문자열을 값으로 갖는다.
// ES6: name 프로퍼티는 함수 객체를 가리키는 변수 이름을 값으로 갖는다.
console.log(anonymousFunc.name); // anonymousFunc

// 함수 선언문(Function declaration)
function bar() {}
console.log(bar.name); // bar
```

### 18.2.5 __proto__ 접근자 프로퍼티
모든 객체는 객체지향의 상속을 구현하는 프로토타입 객체를 가리키는 `[[Prototype]]` 내부 슬롯을 갖는다. `__proto__`는 해당 내부 슬롯이 가리키는 프로토타입 객체에 접근하기 위해 사용하는 접근자 프로퍼티다.

```js
const obj = { a: 1 };

// 객체 리터럴 방식으로 생성한 객체의 프로토타입 객체는 Object.prototype이다.
console.log(obj.__proto__ === Object.prototype); // true

// 객체 리터럴 방식으로 생성한 객체는 프로토타입 객체인 Object.prototype의 프로퍼티를 상속받는다.
// hasOwnProperty 메서드는 Object.prototype의 메서드다.
// 상속받은 프로퍼티인 경우에 false를 반환한다.
console.log(obj.hasOwnProperty('a'));         // true
console.log(obj.hasOwnProperty('__proto__')); // false
```

### 18.2.6 prototype 프로퍼티
생성자 함수로 호출할 수 있는 함수 객체(constructor)만이 소유하는 프로퍼티, 생성자 함수가 생성할 인스턴스의 프로토타입 객체를 가리킨다.
