# 17장 생성자 함수에 의한 객체 생성
## 17.1 Object 생성자 함수
`new` 연산자와 함께 `Object` 생성자 함수를 호출하면 빈 객체를 생성하여 반환한다. 이후, 프로퍼티 메서드를 추가
```js
// 빈 객체의 생성
const person = new Object();

// 프로퍼티 추가
person.name = 'Lee';
person.sayHello = function () {
  console.log('Hi! My name is ' + this.name);
};

console.log(person); // {name: "Lee", sayHello: ƒ}
person.sayHello(); // Hi! My name is Lee
```
이외에도 String, Number, Boolean, Function, Array, Date, RegExp, Promise 등의 빌트인 생성자 함수 제공

## 17.2 생성자 함수
### 17.2.1 객체 리터럴에 의한 객체 생성 방식의 문제점
객체 리터럴에 의한 객체 생성은 직관적이지만, 동일한 프로퍼티를 갖는 객체를 여러 개 생성하는 경우, 같은 프로퍼티를 중복해서 작성하기 때문에 비효율적이다.
```js
const circle1 = {
  radius: 5,
  getDiameter() {
    return 2 * this.radius;
  }
};
const circle2 = {
  radius: 10,
  getDiameter() {
    return 2 * this.radius;
  }
};
```
### 17.2.2 생성자 함수에 의한 객체 생성 방식의 장점
생성자 함수를 사용하는 방법은 객체(인스턴스)를 생성하기 위한 템플릿(클래스)처럼 생성자 함수를 사용하여 프로퍼티 구조가 동일한 객체 여러 개를 간편하게 생성할 수 있다.
- 형식이 있는 클래스 기반 객체지향 언어의 생성자와 달리 일반 함수와 동일한 방법으로 정의하고, **new 연산자와 함께 호출하면 해당 함수는 생성자 함수로 동작**
```js
  // 생성자 함수
  function Circle(radius) {
    // 초기값 할당 코드
    // 생성자 함수 내부의 this는 생성자 함수가 생성할 인스턴스를 가리킨다.
    this.radius = radius;
    this.getDiameter = function () {
      return 2 * this.radius;
    };
  }

// 인스턴스의 생성
const circle1 = new Circle(5);  // 반지름이 5인 Circle 객체를 생성
const circle2 = new Circle(10); // 반지름이 10인 Circle 객체를 생성

console.log(circle1.getDiameter()); // 10
console.log(circle2.getDiameter()); // 20
```
### 17.2.3 생성자 함수의 인스턴스 생성 과정
#### 함수에서 수행하는 것
- 프로퍼티 구조가 동일한 인스턴스를 생성하기 위한 템플릿으로서 동작하여 인스턴스를 생성 (JS 엔진에서 암묵적으로 인스턴스를 생성하고 반환)
- 생성된 인스턴스를 초기화 (인스턴스 프로퍼티 추가 및 초기값 할당, 개발자가 작성하는 코드)

#### 인스턴스 생성 과정
1. **인스턴스 생성과 this 바인딩** (런타임 이전에 실행)
  - `new` 연산자를 사용하여 함수를 호출하면, 암묵적으로 빈 객체(인스턴스) 생성
  - 인스턴스는 `this`에 바인딩(생성자 함수 내부의 this는 해당 인스턴스를 가리킴)
2. **인스턴스 초기화**
- 생성자 함수 내부 코드를 실행, `this`에 바인딩되는 인스턴스에 프로퍼티나 메서드 추가
3. **인스턴스 반환**
- 함수 내부의 처리가 끝나면 완성된 인스턴스가 바인딩 된 `this`를 반환
- 명시적으로 객체를 반환하면 암묵적인 `this` 반환이 무시됨
- 명시적으로 원시 값을 반환하면 `this`가 반환됨
- 따라서 this가 아닌 명시적인 `return`은 지양해야한다.
```js
function Circle(radius) {
  // 1. 암묵적으로 인스턴스가 생성되고 this에 바인딩된다.

  // 2. this에 바인딩되어 있는 인스턴스를 초기화한다.
  this.radius = radius;
  this.getDiameter = function () {
    return 2 * this.radius;
  };

  // 3. 암묵적으로 this를 반환한다.
  // 명시적으로 원시값을 반환하면 원시값 반환은 무시되고 암묵적으로 this가 반환된다.
  return 100;
  // 명시적으로 객체를 반환하면 암묵적인 this 반환이 무시하고 해당 객체를 반환한다.
  return {};
}

// 인스턴스 생성. Circle 생성자 함수는 암묵적으로 this를, 명시적으로는 반환한 객체를 반환한다.
const circle = new Circle(1);
console.log(circle); // Circle {radius: 1, getDiameter: ƒ}
```

### 17.2.4 내부 메서드 \[[Call]] 과 \[[Construct]]
함수는 객체이지만 '호출', '객체 생성자'의 기능을 가지고 있어 일반 객체와는 다르다.
함수 객체는 다음과 같은 슬롯과 메서드를 가진다.
- 일반 객체의 내부 슬롯, 내부 메서드
- 함수 객체를 위한 `[[Environment]]`, `[[[FormalParameters]]]`등의 내부 슬롯
- 일반 함수로서 호출하기 위한 내부 메서드 `[[Call]]`
  - 해당 메서드를 갖는 함수 객체를 `callable`이라 한다.
- 생성자 함수로서 호출하기 위한 내부 메서드 `[[Construct]]`
  - `new` 연산자와 함께 생성자 함수를 호출하면 호출됨
  - 해당 메서드를 갖는 생성자 함수를 `constructor` 갖지 않는 일반 함수를 `non-constructor`라고 한다.
```js
function foo() {}

// 일반적인 함수로서 호출: [[Call]]이 호출된다.
foo();
// 생성자 함수로서 호출: [[Construct]]가 호출된다.
new foo();
```
### 17.2.5 constructor와 non-constructor의 구분
함수가 어디에(객체 프로퍼티 or 변수 식별자) 할당되어 있는지에 따라 메서드인지를 판단하는게 아니라, 함수 정의 방식에 따라 constructor와 non-constructor가 구분된다.

- constructor: 함수 선언문, 함수 표현식, 클래스(클래스도 함수)
- non-constructor: 메서드(ES6 메서드 축약 포현), 화살표 함수
```js
// 일반 함수 정의: 함수 선언문, 함수 표현식
function foo() {}
const bar = function () {};
// 프로퍼티 x의 값으로 할당된 것은 일반 함수로 정의된 함수다. 이는 메서드로 인정하지 않는다.
const baz = {
  x: function () {}
};

// 일반 함수로 정의된 함수만이 constructor이다.
new foo();   // -> foo {}
new bar();   // -> bar {}
new baz.x(); // -> x {}

// 화살표 함수 정의
const arrow = () => {};

new arrow(); // TypeError: arrow is not a constructor

// 메서드 정의: ES6의 메서드 축약 표현만을 메서드로 인정한다.
const obj = {
  x() {}
};

new obj.x(); // TypeError: obj.x is not a constructor
```
### 17.2.6 new 연산자
new 연산자와 함께 함수를 호출하면 해당 함수는 생성자 함수로 동작한다(`[[Call]]`이 아니라 `[[Construct]]`가 호출된다.) 단, new 연산자와 함께 호출하는 함수는 `constructor`이어야 한다. (함수 선언문, 함수 표현식, 클래스)

- 원시 값을 반환하는 일반 함수를 `new`와 함께 호출하면 반환문이 무시되고 빈 객체가 생성된다.
- 객체를 반환하는 일반 함수를 `new`와 함께 호출하면 해당 객체가 반환된다.
- `this`를 내부에서 사용하며 반환하지 않는 함수(생성자 함수)를 `new`없이 호출하면 일반 함수로서 호출된다
    - `undefined`가 반환된다.
    - 일반 함수 내부의 this는 전역 객체 `window`를 가리킨다.
### 17.2.7 new.target
- ES6에서 지원하는 메타 프로퍼티 (constructor인 모든 함수 내부에서 암묵적인 지역 변수로 사용된다.)
- 해당 함수가 `new`연산자와 함께 호출되었는지 확인할 수 있다.
- `new`와 함께 생성자 함수로서 호출되면 함수 내부의 `new.target`은 함수 자신을 가리킨다. (`new` 연산자 없이 호출된 함수 내부에서는 `undefined`)

```js
// 생성자 함수
function Circle(radius) {
  // 이 함수가 new 연산자와 함께 호출되지 않았다면 new.target은 undefined다.
  if (!new.target) {
    // new 연산자와 함께 생성자 함수를 재귀 호출하여 생성된 인스턴스를 반환한다.
    return new Circle(radius);
  }

  this.radius = radius;
  this.getDiameter = function () {
    return 2 * this.radius;
  };
}

// new 연산자 없이 생성자 함수를 호출하여도 new.target을 통해 생성자 함수로서 호출된다.
const circle = Circle(5);
console.log(circle.getDiameter());  // 10
```

- `new.target`은 IE에서 지원하지 않으므로, `this`의 값을 이용한 스코프 세이프 생성자 패턴을 사용할 수 있다 (p. 247)
- 대부분의 빌트인 생성자 함수는 스코프 세이프 패턴을 사용해서 `new` 연산자와 함께 호출되었는지 확인한 후 적절한 값을 반환하기 때문에, `new` 연산자를 생략해도 동일하게 동작한다. (String, Number, Boolean은 매개변수의 값을 데이터 타입을 변환하여 그대로 반환한다.)
