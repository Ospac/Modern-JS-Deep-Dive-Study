# 12장 함수
## 12.1 함수란?
**함수**는 일련의 과정을 문으로 구현하고 코드 블록으로 감싸서 하나의 실행단위로 정의 한 것이다. 함수 정의를 통해 생성되고, 함수 호출을 통해 실행 결과를 반환한다.
## 12.2 함수를 사용하는 이유
코드의 **재사용**을 위하여 필수적이고, 재사용을 하면, 다음 두가지 이점이 파생된다. 
- 유지보수의 편의성: 중복된 코드는 여러 코드 베이스에 접근하지 않고, 한번에 수정할 수 있다.
- 가독성: 실행 흐름을 압축한 함수명으로 기능을 쉽게 파악할 수 있게 한다.
## 12.3 함수 리터럴
자바스크립트의 함수는 **객체 타입의 값**이다. 따라서 함수 리터럴을 생성하여 변수에 할당할 수 있다.
```js
// 변수에 함수 리터럴을 할당
var f = function add(x, y) {
  return x + y;
};
```
## 12.4 함수 정의
함
수 정의에는 4가지 방식이 있다.
- 함수 선언문
- 함수 표현식
- Function 생성자 함수
- 화살표 함수(ES6)
### 12.4.1 함수 선언문
```js
// 함수 선언문
function add(x, y) {
  return x + y;
}
```
함수 선언문은 표현식이 아닌 문이다. 변수에 할당할 수 없다.
하지만, 변수에 할당이 가능한데, 해당 코드블록은 중의적인 맥락에서 **함수 선언문**이 아니라 **함수 리터럴**로 해석된다.
```js
// 함수 선언문은 표현식이 아닌 문이므로 변수에 할당할 수 없다.
// 하지만 함수 선언문이 변수에 할당되는 것처럼 보인다.
var variable = function add(x, y) {
  return x + y;
};

// 함수 호출
console.log(variable(2, 5)); // 7
```
함수는 함수 이름으로 호출하는 것이 아니라 **함수 객체를 가리키는 식별자**로 호출한다,
자바스크립트 엔진은 생성된 함수를 호출하기 위해서, 함수 이름과 동일한 이름의 식별자를 암묵적으로 생성하고, 거기에 함수 객체를 할당한다. 다음과 같은 형태로 변환되는 것.
```js
function add(x, y){
  return x + y;
}
// ->
var add = function add(x, y) {
  return x + y;
};
```
이는 함수 표현식의 형태지만, 함수 선언문과 함수 표현식이 동일하게 동작하는 것은 아니다.

### 12.4.2 함수 표현식
값의 성질을 갖는 객체를 일급 객체라 하고, 자바스크립트의 함수는 일급 객체다. 따라서 함수 리터럴로 생성한 함수 객체를 변수에 할당할 수 있다. 이러한 정의 방식을 **함수 표현식**<sup>function expression</sup>이라 한다.
- 익명 함수: 함수 리터럴의 함수 이름은 생략할 수 있다. (함수 객체를 가리키는 변수로 호출되기 때문)
```js
var add = function (x, y) {
  return x + y;
};
console.log(add(2, 5)); // 7
```
### 12.4.3 함수 생성 시점과 함수 호이스팅
```js
// 함수 참조
console.dir(add); // ƒ add(x, y)
console.dir(sub); // undefined

// 함수 호출
console.log(add(2, 5)); // 7
console.log(sub(2, 5)); // TypeError: sub is not a function

// 함수 선언문
function add(x, y) {
  return x + y;
}

// 함수 표현식
var sub = function (x, y) {
  return x - y;
};
```
함수 선언문과 함수 표현식은 생성시점이 다르다.
#### 함수 호이스팅
**함수 선언문**으로 선언된 함수는 런타임 이전에 함수 객체가 먼저 생성되고 식별자가 생성되어 객체가 할당된다. 따라서, 런타임에는 함수 선언문 이전에 함수를 참조하고 호출할 수 있다.
#### 변수 호이스팅
**함수 표현식**으로 선언된 함수는 `var` 변수 호이스팅에 영향을 받는다. 따라서, 변수의 선언부가 먼저 평가되며 `undefined`로 초기화된다. 런타임에는 함수 표현식 이전에 함수를 호출하면, `TypeError`가 나온다.
### 12.4.4 Function 생성자 함수
```js
var add = new Function('x', 'y', 'return x + y');
console.log(add(2, 5)); // 7
```
`new` 연산자와 `Function` 생성자 함수를 이용하여 함수를 선언하는 방식
클로저를 생성하지 않는 특징이 있어 권장되지 않는다.
### 12.4.5 화살표 함수
```js
// 화살표 함수
const add = (x, y) => x + y;
console.log(add(2, 5)); // 7
```
ES6에서 도입된 화살표 함수<sup>arrow function</sup> 항상 익명 함수로 정의한다.
함수 선언문보다 내부 기능이 간략화되어 있다. (26.3 화살표 함수에서 자세히)
## 12.5 함수 호출
### 12.5.1 매개변수와 인수
함수를 정의할때 **매개변수**<sup>parameter</sup>를 선언하며, 함수를 호출할때 **인수**<sup>argument</sup>를 전달한다.
- 매개변수는 함수 내부에서 변수와 동일하게 취급된다.
- 매개변수는 일반 변수와 마찬가지로 `undefined`로 초기화, 인수가 순서대로 할당된다.
- 매개변수의 스코프는 함수 내부이다.
- 인수가 부족해서 할당되지 않은 매개변수의 값은 `undefined`
- 인수가 매개변수보다 더 많은 경우, 초과된 인수는 `arguments` 객체의 프로퍼티로 보관된다. (18.2.1)

### 12.5.2 인수 확인
인수의 타입, 갯수 체크
- ES6에서 도입된 매개변수 기본값 활용(매개변수에 인수를 전달하지 않았을 경우, `undefined`를 전달한 경우에만 유효)
### 12.5.3 매개변수의 최대 개수
- 이상적인 함수는 한 가지 일 해야 하며 가급적 작게 만들어야 한다.
- 최대 3개 이상을 넘지 않는 것을 권장, 그 이상이 필요하다면 객체를 전달하는 것이 유리
- 객체를 전달할 경우, 객체를 내부에서 변경시, 외부의 객체가 변경되는 부수효과 발생 (12.6)

### 12.5.4 반환문
`return` 키워드와 표현식(반환 값)으로 이뤄진 반환문을 사용해 실행 결과를 함수 외부로 반환

함수 호출은 표현식이다. `return` 키워드가 반환한 표현식의 평가 결과, 반환값으로 평가된다.
`return`을 생략하거나, `return` 뒤 표현식을 생략하면 `undefined`가 반환된다,
## 12.6 참조에 의한 전달과 외부 상태의 변경
```js
// 값을 변경하는 함수
// 매개변수 primitive는 원시 값을 전달받고, 매개변수 obj는 객체를 전달받는다.
function changeVal(primitive, obj) {
  primitive += 100;
  obj.name = 'Kim';
}

// 외부 상태
var num = 100;
var person = { name: 'Lee' };

console.log(num); // 100
console.log(person); // {name: "Lee"}

// 원시 값은 값 자체가 복사되어 전달되고 객체는 참조 값이 복사되어 전달된다.
changeVal(num, person);

// 원시 값은 원본이 훼손되지 않는다.
console.log(num); // 100

// 객체는 원본이 훼손된다.
console.log(person); // {name: "Kim"}
```
- 원시 타입의 인수는 값 자체가 복사되어 매개변수에 전달되기 때문에, 외부의 원시 값 원본을 변경하는 어떤 부수 효과도 발생하지 않는다.
- 객체 타입의 인수는 **참조 값**(주소)이 복사되어 매개변수에 전달되기 때문에, 참조 값을 통해 객체를 변경하면, 원본도 같이 변경된다. 함수 내부에서 외부 객체를 변경하는 **부수효과가 발생**한다.

부수효과가 있는 코드는 복잡성을 증가하고, 가독성을 해친다. 내부 동작에서 외부 상태가 변하는지 알기가 어렵기 때문이다. 다음과 같은 해결책들이 있다.
- 옵저버 패턴으로 객체를 참조하는 모든 코드에서 알림을 받고 대응
- 객체를 불변 객체로 만들어 사용 (16.5.4)
- 객체의 상태 변화가 필요한 경우, 깊은 복사를 통해 새로운 객체를 생성하여 값 수정
