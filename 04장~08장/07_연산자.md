# 07장 연산자

**연산자**<sup>operator</sup> 하나 이상의 표현식을 대상으로 산술, 할당, 비교, 논리, 타입, 지수 연산등을 수행해 하나의 값을 만든다.
피연산자가 "값"이라는 명사 역할을 한다면 연산자는 "피연산자를 연산하여 새로운 값을 만든다"라는 **동사의 역할**을 한다고 볼 수 있다.

## 7.1 산술 연산자

산술 연산자<sup>arithmetic operator</sup>는 피연산자를 대상으로 계산을 수행해 새로운 숫자 값을 만든다. 산술 연산이 불가능한 경우 NaN을 반환한다.

- 이항 산술 연산자 `+`, `-`, `\*`, `/`, `%`

  - 피연산자의 값을 변경하는 부수효과가 없음
  - `+` 이항 연산자는 피연산자 중 하나 이상이 문자열인 경우 문자열 연결 연산자로 동작한다. ex. `'1' + 2` -> '12'

- 단항 산술 연산자
  - 증감 연산자 (`++`, `--`)
    - 전위, 후위의 차이가 존재
    - 피연산자의 값을 변경하는 부수효과 존재
  - `+`, `-`
    - `+` 단항 연산자는 피연산자를 숫자 타입으로 변환하여 반환
      - undefined는 숫자로 타입 변환되지 않는다 ex. `+undefined` -> NaN
    - `-` 단항 연산자는 피연산자를 숫자 타입으로 변환하고, 부호를 반전한 값을 반환

## 7.2 할당 연산자

할당 연산자<sup>assignment operator</sup>는 우항에 있는 피연산자의 평가 값을 좌항에 있는 변수에 할당한다. 할당 연산자가 쓰인 할당문은 피연산자 값이 변하는 **부수효과**가 있고, 동시에 **값으로 평가 되는 표현식**이다.

```js
// 연쇄할당 예시
var a, b, c;
a = b = c = 0;
```

## 7.3 비교 연산자

**비교 연산자**<sup>comparison operator</sup>는 두 피연산자를 비교 한 다음 그 결과를 Boolean 값으로 반환한다.

1. **동등 비교**<sup>loose equality</sup>연산자는 피연산자를 비교할 때 먼저 **암묵적 타입 변환**을 통해 타입을 일치시킨 후 같은 값인지 비교한다.

```js
5 == "5"; // true

//결과를 예측하기 어렵고 실수하기 쉽다.
"0" == ""; // -> false
0 == ""; // -> true
0 == "0"; // -> true
false == "false"; // -> false
false == "0"; // -> true
false == null; // -> false
false == undefined; // -> false
```

2. **일치 비교**<sup>strict equality</sup>연산자는 피연산자가 타입도 같고 값도 같은 경우 한하여 true를 반환한다. 암묵적 타입 변환을 하지 않고 값을 비교해서, 예측하기 쉽다.

단, 숫자가 NaN인지 체크하는 경우 이해하기 어려운 결과가 나온다.

```js
NaN === NaN -> false

// 빌트인 함수 isNaN 사용이 권장된다.
isNaN(NaN) -> true
isNaN(10) -> false
```

## 7.4 삼항 조건 연산자

삼항 조건 연산자<sup>ternary operator</sup>는 물음표 앞 첫번째 연산자를 암묵적 타입변환 후, 불리언 타입의 값으로 평가하여 참, 거짓에 따라 두번째, 세번째 피연산자를 반환하는 연산자이며, 해당 표현식은 값으로 평가할 수 있다.

```js
var result = x % 2 ? "홀수" : "짝수";
```

## 7.5 논리 연산자

논리 부정(!) 연산자는 암묵적 타입변환을 이용하여 불리언 타입으로 변환 할 수 있다.

```js
!0; -> true
!!'Hello' -> true
```

## 7.6 쉼표 연산자

쉼표 연산자는 왼쪽부터 차례대로 피연산자를 평가하고, 마지막 피연산자의 평가 결과를 반환한다.

```js
var x, y, z;

x = 1, y = 2, z = 3; //3
```

## 7.7 그룹 연산자

그룹 연산자는 소괄호 `()`로 피연산자를 감싸서 해당 표현식을 가장 먼저 평가한다. 연산자 우선순위가 가장 높다.

```js
10 * (2 + 3); // -> 50
```

## 7.8 typeof 연산자
`typeof` 연산자는 피연산자의 데이터 타입을 문자열로 반환한다.

```js
typeof ''              // -> "string"
typeof 1               // -> "number"
typeof NaN             // -> "number"
typeof true            // -> "boolean"
typeof undefined       // -> "undefined"
typeof Symbol()        // -> "symbol"
typeof null            // -> "object"
typeof []              // -> "object"
typeof {}              // -> "object"
typeof new Date()      // -> "object"
typeof /test/gi        // -> "object"
typeof function () {}  // -> "function"
```
null은 `object`로 반환한다. 따라서, 값이 null 타입인지 확인할 때는 typeof 연산자가 아니라 일치 연산자를 사용해야한다.
```js
var foo = null;

typeof foo === null; // -> false
foo === null;        // -> true
```
선언하지 않은 식별자를 typeof로 연산할때도 ReferenceError가 발생하지 않고 undefined를 반환하는 이슈가 있다.

```js
// undeclared 식별자를 선언한 적이 없다.
typeof undeclared; // -> undefined
```

## 7.9 지수 연산자
ES7에서 도입, 좌항을 밑<sup>base</sup>으로 우항을 지수<sup>exponent</sup>로 거듭 제곱하여 값을 반환한다.
```js
2 ** 2;   // -> 4
2 ** 0;   // -> 1
2 ** -2;  // -> 0.25

//할당 연산자와 같이 사용
var num 5;
num **= 2; // -> 25
```

## 7.11 연산자의 부수효과
일부 연산자는 다른 코드에 영향을 주는 부수 효과가 있다.
- 할당 연산자(=)
```js
var x;

// 할당 연산자는 변수 값이 변하는 부수 효과가 있다.
// 이는 x 변수를 사용하는 다른 코드에 영향을 준다.
x = 1;
console.log(x); // 1
```
- 증감 연산자(++/--)
```js
// 증가/감소 연산자(++/--)는 피연산자의 값을 변경하는 부수 효과가 있다.
// 피연산자 x의 값이 재할당되어 변경된다. 이는 x 변수를 사용하는 다른 코드에 영향을 준다.
x++;
console.log(x); // 2
```
- delete 연산자
```js
var o = { a: 1 };

// delete 연산자는 객체의 프로퍼티를 삭제하는 부수 효과가 있다.
// 이는 o 객체를 사용하는 다른 코드에 영향을 준다.
delete o.a;
console.log(o); // {}
```
