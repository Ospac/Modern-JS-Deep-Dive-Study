# 06장 데이터 타입

자바스크립트의 모든 값은 **데이터 타입**<sup>data type</sup>(값의 종류)을 갖는다. 데이터 타입은 원시 타입<sup>primitive type</sup>과 객체 타입<sup>object/reference type</sup>으로 분류된다.

가령, 숫자 타입의 값 1과 문자열 타입의 값 1은 비슷해 보이지만 생성한 목적과 용도가 전혀 다른 값이다. 확보한 메모리 공간의 크기도 다르고 읽어들여 해석하는 방식도 다르다.

## 6.1 숫자 타입

자바스크립트는 독특하게 하나의 숫자 타입만 존재한다.
ECMAScript 사양에 따라, 숫자 타입의 값은 배정밀도 64비트 부동소수점 형식을 따른다.

1. **모든 수를 실수로 처리**한다.

```js
console.log(1 === 1.0); // true
console.log(4 / 2); // 2
console.log(3 / 2); // 1.5
```

2. 2진수, 8진수, 16진수를 참조하면 모두 10진수로 해석된다.

```js
var binary = 0b01000001; // 2진수
var octal = 0o101; // 8진수
var hex = 0x41; // 16진수

// 표기법만 다를 뿐 모두 같은 값이다.
console.log(binary); // 65
console.log(octal); // 65
console.log(hex); // 65
console.log(binary === octal); // true
console.log(octal === hex); // true
```

3. 숫자타입은 다음 세가지 값도 표현

- Infinity
- -Infinity
- NaN (not-a-number)

```js
// 숫자 타입의 세 가지 특별한 값
console.log(10 / 0); // Infinity
console.log(10 / -0); // -Infinity
console.log(1 * "String"); // NaN
```

## 6.2 문자열 타입

- 16비트 유니코드 문자(UTF-16)의 집합. 대부분의 문자를 표현할 수 있다.
- 자바스크립트의 문자열은 변경 불가능한 값<sup>immutable value</sup> (11.1.2)

## 6.3 템플릿 리터럴

ES6부터 도입된 새로운 문자열 표기법. 다음 세가지 기능이 있다.

- 멀티라인 문자열
  - 이스케이프 시퀀스 없이도 줄바꿈이 허용된다.
- 표현식 삽입
  - `${ }`로 감싸서 표현식을 삽입할 수 있음.
- 태그드 템플릿
  - 태그 즉 이름표처럼 템플릿 리터럴 앞에 함수가 붙은 경우

```js
function tagFunction(strings: TemplateStringsArray, ...values: any[]) {
  console.log("strings: ", strings);
  console.log("values: ", values);
}
const expression = "foo";
tagFunction`string text ${expression} string text`;
// strings: [ 'string text ', ' string text' ]
// values: [ 'foo' ]
```

## 6.4 불리언 타입

논리적 참, 거짓을 나타내는 `true`, `false`

## 6.5 undefined 타입

자바스크립트 엔진이 변수를 초기화할때 사용하는 값, 선언된 변수는 암묵적으로 `undefined`로 초기화 된다.

## 6.6 null 타입

변수에 값이 없다는 것을 의도적으로 명시할 떄 사용한다. 변수가 이전에 참조하던 값을 명시적으로 제거할때 사용한다. (이 경우, `undefined`를 할당하는 것은 권장되지 않음)

## 6.7 심벌 타입

ES6에서 도입된 타입, 심벌<sup>symbol</sup>값은 다른 값과 중복되지 않는 유일무이한 값이다. 이름이 충돌할 위험이 없는 객체의 유일한 프로퍼티 키를 만들기 위해 사용된다.

```js
// Symbol 함수를 호출해 생성
var key = Symbol("key");
console.log(typeof key); // symbol

// 객체 생성
var obj = {};

// 이름이 충돌할 위험이 없는 유일무이한 값인 심벌을 프로퍼티 키로 사용한다.
obj[key] = "value";
console.log(obj[key]); // value
```

## 6.8 객체 타입

자바스크립트는 객체 기반 언어이며, 자바스크립트를 이루고 있는 거의 모든 것이 객체
(11장 원시 값과 객체의 비교에서 자세히)

## 6.9 데이터 타입의 필요성

1. 데이터 타입에 따라 값을 저장할때 확보해야할 메모리 크기가 다르다. (ex. `number` type은 8byte)
2. 값을 참조할 때 한 번에 읽어 들여야 할 메모리 공간의 크기를 결정하기 위해서(참조하는 주소에서 1byte를 읽을 것이냐, 8byte(number)를 읽을 것이냐)
3. 읽어 들인 값(2진수)를 어떻게 해석할 것인지 결정하기 위해서
   - 같은 값도 데이터 타입에 따라서 다르게 해석됨 (ex. 0100 0001 -> 'A', 65)

## 6.10 동적 타이핑

자바스크립트는 **동적 타이핑**<sup>dynamic/weak type</sup> 언어이다. 따라서, 어떤 데이터 타입의 값이라도 변수에 자유롭게 할당할 수 있다. 자바스크립트의 변수는 선언이 아닌 **할당에 의해 타입이 결정**(타입 추론)된다. 이러한 특징을 **동적 타이핑**<sup>dynamic typing</sup>이라고 한다.

- 변수는 타입을 갖지 않는다.
- 값은 타입을 갖는다.
- 변수에 할당되어 있는 값에 의해 변수의 타입이 동적으로 결정된다.

동적 타입 언어는 유연성은 높지만 신뢰성은 떨어진다.

반면, 변수를 선언할 때 데이터 타입을 같이 선언하는 **정적 타입**<sup>static/strong type</sup> 언어는 변수 타입을 변경할 수 없으며, 변수에 선언한 타입에 맞는 값만 할당할 수 있다. 또한, 컴파일 시점에, 타입체크를 수행한다.
