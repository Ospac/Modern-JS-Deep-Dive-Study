# 41.1 호출 스케줄링

일정 시간이 경과된 후에 호출되도록 함수 호출을 예약하려면 타이머 함수를 사용하는데, 이를 호출 스케줄링<sup>scheduling a call</sup> 이라 한다.
- 자바스크립트는 타이머를 생성하는 타이머 함수 `setTimeout`과 `setInterval`, 이를 제거할 수 있는 `clearTimeout`, `clearInterval`을 쌍으로 제공한다.
- 타이머 함수는 빌트인 객체가 아니라 실행 환경에서 제공하는 호스트 객체다.
## 41.2 타이머 함수
### 41.2.1 `setTimeout` / `clearTimeout`

`setTimeout` 함수는 두 번째 인수로 전달받은 시간(ms, 1 / 1000초) 이후 첫 번째 인수로 전달받은 콜백 함수가 한번 호출되는 형태로 동작하는 타이머를 생성한다.

```js
const timeoutId = setTimeout(func|code[, delay, param1, param2, ...]);
```

![setTimeout](https://github.com/user-attachments/assets/8e335abf-81ed-41a6-9b09-7d8137285638)

```js
// 1초(1000ms) 후 타이머가 만료되면 콜백 함수가 호출된다.
// setTimeout 함수는 생성된 타이머를 식별할 수 있는 고유한 타이머 id를 반환한다.
const timerId = setTimeout(() => console.log('Hi!'), 1000);

// setTimeout 함수가 반환한 타이머 id를 clearTimeout 함수의 인수로 전달하여 타이머를
// 취소한다. 타이머가 취소되면 setTimeout 함수의 콜백 함수가 실행되지 않는다.
clearTimeout(timerId);
```

### 41.2.2 `setInterval` / `clearInterval`

`setInterval` 함수는 두 번째 인수로 전달받은 시간(ms, 1 / 1000초)으로 반복 동작하는 타이머를 생성한다. 타이머가 만료될 때마다 첫 번째 인수로 전달받은 콜백 함수가 반복 호출된다.
이는 타이머가 취소될 때까지 계속된다.

```js
const timeerId = setInterval(func|code[, delay, param1, param2, ...]);
```

```js
let count = 1;

// 1초(1000ms) 후 타이머가 만료될 때마다 콜백 함수가 호출된다.
// setInterval 함수는 생성된 타이머를 식별할 수 있는 고유한 타이머 id를 반환한다.
const timeoutId = setInterval(() => {
  console.log(count); // 1 2 3 4 5
  // count가 5이면 setInterval 함수가 반환한 타이머 id를 clearInterval 함수의
  // 인수로 전달하여 타이머를 취소한다. 타이머가 취소되면 setInterval 함수의 콜백 함수가
  // 실행되지 않는다.
  if (count++ === 5) clearInterval(timeoutId);
}, 1000);
```
## 41.3 디바운스와 스로틀
`scroll`, `resize`, `input`, `mousemove` 같은 이벤트는 짧은 시간 간격으로 연속으로 발생하기 때문에, 이러한 이벤트에 바인딩한 핸들러는 과도하게 호출되어 성능에 문제를 일으킬 수 있다.
디바운스와 스로틀은 짧은 시간에 연속해서 발생하는 **이벤트를 그룹화하여 이벤트 핸들러의 호출 횟수를 줄이는** 프로그래밍 기법이다.

타이머 함수를 활용하여 디바운스와 스로틀을 구현한다.
### 41.3.1 디바운스

디바운스<sup>debounce</sup>는 이벤트를 그룹화한다. 디바운스가 적용된 이벤트 핸들러는 짧은 시간 간격으로 이벤트가 연속해서 발생하면 콜백 함수를 호출하지 않다가, 일정 시간이 경과된 이후 한번만 호출되도록 한다.

#### `input` 이벤트의 예시
```html
<input type="text">
<div class="msg"></div>
<script>
const $input = document.querySelector('input');
const $msg = document.querySelector('.msg');

const debounce = (callback, delay) => {
  let timerId;
  // debounce 함수는 timerId를 기억하는 클로저를 반환한다.
  return (...args) => {
	// delay가 경과하기 이전에 이벤트가 발생하면 이전 타이머를 취소하고
	// 새로운 타이머를 재설정한다.
	// 따라서 delay보다 짧은 간격으로 이벤트가 발생하면 callback은 호출되지 않는다.
	if (timerId) clearTimeout(timerId);
	timerId = setTimeout(callback, delay, ...args);
  };
};

// debounce 함수가 반환하는 클로저가 이벤트 핸들러로 등록된다.
// 300ms보다 짧은 간격으로 input 이벤트가 발생하면 debounce 함수의 콜백 함수는
// 호출되지 않다가 300ms 동안 input 이벤트가 더 이상 발생하면 한 번만 호출된다.
$input.oninput = debounce(e => {
  $msg.textContent = e.target.value;
}, 300);
</script>
```
- `input` 이벤트는 사용자가 텍스트 입력 필드에 값을 입력할 때마다 연속해서 발생한다. 이때 이벤트 핸들러 호출마다 HTTP 요청을 한다면 서버에 부담을 주게 될 것이다.
- 따라서, 사용자가 입력을 완료했을때 한 번만 요청하는 것이 효율적이다.
- 디바운스 기법에서는 일정 시간 동안 텍스트 입력 필드에 값이 입력되지 않으면 입력이 완료된 것으로 간주한다.
- `delay`보다 짧은 간격으로 이벤트가 발생하면, 클로저 변수인 `timerId`를 참조하여 이전 타이머를 취소하고 새로운 타이머를 재설정하여 `callback` 호출을 스킵한다.
- `clearTimeout`이 호출되지 않고 `delay` 시간이 지나면, `callback`이 정상적으로 호출된다.



![41-2](https://github.com/user-attachments/assets/d20693ea-4616-41f2-be05-44e03126412a)

- 디바운스는 `resize` 이벤트 처리나, `input` 요소의 값으로 `ajax`를 요청하는 입력 필드 자동완성 UI 구현, 버튼 중복 클릭 방지 처리 등에 사용한다.
- 실무에서는 `Underscore`의 `debounce`함수나 `Lodash`의 `debounce` 함수를 사용하는 것을 권장
### 41.3.2 스로틀

스로틀 <sup>throttle</sup>은 짧은 시간 간격으로 이벤트가 연속해서 발생하더라도 **일정 시간 간격으로 이벤트가 최대 한 번만 호출**되도록 한다.
따라서 일정시간 단위로 이벤트 핸들러가 호출되도록 호출 주기를 만든다.

#### `scroll` 이벤트가 짧은 시간 간격으로 연속해서 발생하는 예시

```js
  <div class="container">
    <div class="content"></div>
  </div>
  <div>
    일반 이벤트 핸들러가 scroll 이벤트를 처리한 횟수:
    <span class="normal-count">0</span>
  </div>
  <div>
    스로틀 이벤트 핸들러가 scroll 이벤트를 처리한 횟수:
    <span class="throttle-count">0</span>
  </div>

  <script>
    const $container = document.querySelector('.container');
    const $normalCount = document.querySelector('.normal-count');
    const $throttleCount = document.querySelector('.throttle-count');

    const throttle = (callback, delay) => {
      let timerId;
      // throttle 함수는 timerId를 기억하는 클로저를 반환한다.
      return (...args) => {
        // delay가 경과하기 이전에 이벤트가 발생하면 아무것도 하지 않다가
        // delay가 경과했을 때 이벤트가 발생하면 새로운 타이머를 재설정한다.
        // 따라서 delay 간격으로 callback이 호출된다.
        if (timerId) return;
        timerId = setTimeout(() => {
          callback(...args);
          timerId = null;
        }, delay);
      };
    };

    let normalCount = 0;
    $container.addEventListener('scroll', () => {
      $normalCount.textContent = ++normalCount;
    });

    let throttleCount = 0;
    // throttle 함수가 반환하는 클로저가 이벤트 핸들러로 등록된다.
    $container.addEventListener('scroll', throttle(() => {
      $throttleCount.textContent = ++throttleCount;
    }, 100));
  </script>
```

- `throttle` 함수가 반환한 함수는 `throttle` 함수에 전달한 `delay` 시간이 경과 하기 이전에 이벤트가 발생하면 아무것도 하지 않는다.
- `delay` 시간이 경과했을 때 이벤트가 발생하면 콜백 함수를 호출하고 새로운 타이머를 재설정한다.
- 따라서 `delay` 시간 간격으로 콜백 함수가 호출된다.

![41-3](https://github.com/user-attachments/assets/608aa9b7-3d08-4647-a07a-1b13e6b32463)

이벤트를 그룹화하여 일정 시간 간격으로 이벤트 핸들러를 호출하는 스로틀은 `scroll` 이벤트 처리나 무한 스크롤 UI 구현등에 사용된다.
- 실무에서는 `Underscore`의 `throttle`함수나 `Lodash`의 `throttle` 함수를 사용하는 것을 권장
