# 13장 스코프
## 13.1 스코프란?
스코프는 식별자가 유효한 범위다. 이를테면 모든 식별자는 자신이 선언된 위치에 따라서 다른 코드가 자신을 참조할 수 있는 유효 범위가 결정된다. 이를 **스코프**라 한다.

- 식별자를 검색할 때 사용하는 규칙
- **네임스페이스** 역할 : 프로그래밍 언어에서는 스코프를 통해 변수 이름의 충돌을 방지하고, 같은 이름의 변수를 사용할 수 있게 한다. 한 스코프 내에서는 식별자는 유일하지만 다른 스코프에는 같은 이름의 식별자를 사용할 수 있다.

## 13.2 스코프의 종류
코드는 전역<sup>global</sup>과 지역<sup>local</sup>으로 구분할 수 있다.
변수는 선언된 위치에 따라 스코프(전역 스코프, 지역 스코프)가 결정된다.
- 전역 변수
  - 코드의 가장 바깥 영역
  - 어디서든 참조 가능
- 지역 변수
  - 함수 내부
  - 자신의 지역 스코프 혹은 하위 지역 스코프에서만 참조 (이외에 참조시 `ReferenceError`)

```js
var x = 'global';

function foo() {
  var x = 'local';
  console.log(x); // ① local
}

foo();

console.log(x); // ② global
```
①의 경우 참조가 모호하지만 (지역변수 x, 전역변수 x 어느쪽을 가리키는지), 자바스크립트 엔진이 스코프 체인을 통해 참조할 변수를 검색하여 결정한다.
## 13.3 스코프 체인
함수는 전역에서 정의될 수 있고, 함수 내부에서 정의될 수 있어서 함수는 중첩될 수 있다.
전역 스코프, outer 함수 스코프, inner 함수 스코프 ...의 계층으로 연결된 것을 **스코프 체인**이라고 한다.

앞선 예시처럼, 스코프가 중첩된 경우 참조가 모호한 경우가 발생한다. 변수를 참조할 때 자바스크립트 엔진은 스코프 체인을 통해, **참조하는 해당 스코프에서 시작하여, 상위 스코프 방향으로 이동하여 변수를 검색한다.** (하위 스코프로 내려가서 검색하지 않는다.)

## 13.4 함수 레벨 스코프 

`var` 키워드로 선언된 변수는 오직 함수의 코드 블록만을 지역 스코프로 인정한다. 이러한 특성을 **함수 레벨 스코프**<sup>function level scope</sup>라 한다.

두가지 예시가 있다.
```js
var x = 1;

if (true) {
  // var 키워드로 선언된 변수는 함수의 코드 블록(함수 몸체)만을 지역 스코프로 인정한다.
  // 함수 밖에서 var 키워드로 선언된 변수는 코드 블록 내에서 선언되었다 할지라도 모두 전역 변수다.
  // 따라서 x는 전역 변수다. 이미 선언된 전역 변수 x가 있으므로 x 변수는 중복 선언된다.
  // 이는 의도치 않게 변수 값이 변경되는 부작용을 발생시킨다.
  var x = 10;
}

console.log(x); // 10
```

```js
var i = 10;

// for 문에서 선언한 i는 전역 변수다. 이미 선언된 전역 변수 i가 있으므로 중복 선언된다.
for (var i = 0; i < 5; i++) {
  console.log(i); // 0 1 2 3 4
}

// 의도치 않게 변수의 값이 변경되었다.
console.log(i); // 5
```
대비 되는 개념으로 `let`과 `const` 키워드로 선언된 변수는 블록 레벨 스코프를 지원한다. (15장)
## 13.5 렉시컬 스코프

```js
var x = 1;

function foo() {
  var x = 10;
  bar();
}

function bar() {
  console.log(x);
}

foo(); // ?
bar(); // ?
```
다음 예제의 실행 결과는 스코프 처리 방식에 따라 결정된다. 
두가지 방식이 있고 자바스크립트는 정적스코프, 렉시컬 스코프를 따른다.
1. 동적 스코프<sup>dynamic scope</sup> : **함수를 어디서 호출**했는지에 따라 함수의 상위 스코프를 결정한다.
    - 함수가 호출되는 시점에서 상위 스코프를 결정한다.
    - `foo()`를 호출하면, `foo` 내부의 `bar()`가 호출되고 `bar`의 상위 스코프는 그 시점에서 `foo`로 결정된다. 따라서 `var x = 10`을 참조한다.
    - 10, 1을 출력
2. 정적 스코프(렉시컬 스코프<sup>lexical scope</sup>) : **함수를 어디서 정의**했는지에 따라 함수의 상위 스코프를 결정한다.
    - 함수 정의가 평가되는 시점에서 상위 스코프가 결정된다.
    - `bar`는 자신이 정의된 스코프. 즉 전역 스코프를 기억한다.
    - `bar`는 어디서 호출되든 상관없이 `var x = 1`을 기억하고 참조한다. (24장 클로저와 관련있음)
    - 1, 1을 출력


함수의 상위 스코프는 함수 정의가 실행될 때 결정된다.
생성된 함수 객체는 이렇게 결정된 상위 스코프를 기억한다.
